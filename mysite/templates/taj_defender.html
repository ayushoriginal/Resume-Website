A sand-boxing mechanism to run malicious / untrusted C and C++ codes. The main objective was to run a C/C++ code whose origin or content is not known as such such codes pose serious threats to system's security.

This project was done as a part of the Operating Systems course during my undergrad between March 2014 to April 2014.

Basic Working
The execution of a typical C program is depicted in one of the images. Our program compiles the given code and prevents any system calls made by the function except read-write from standard input and standard output. For memory allocation we provide a custom implementation of malloc.h. If any system call is detected, the program is terminated with run time error.

Features
The main features of the projects are :

  • SECCOMP was used to prevent malicious system calls
  • Custom implementation of memory allocating functions in C/C++
  • Provides simple interface to load and execute a program in sand-boxed environment via a script
  • Pre-processing the program for commonly known malicious signature
