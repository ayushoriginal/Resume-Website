A graph based storage engine for efficient storage of data in a Relational Database. This project was done as a part of the Minor Project course during my undergrad in 6th semester from February to April 2015.

It is an ambitious research project that aims to minimize the cost of aggregate and join queries in a relational database. It also reduces data redundancy by storing each value in a relation only once.

A relational database is modeled as a graph in our proposed scheme. Columns of each table is represented as a singly linked list. An actual row of a relational database is represented by a node in the main linked list (which represents a table) and the edges from that node to different attribute nodes which store the actual values. In our model, columns can be shared between tables allowing us to model foreign key constraint.

We were able to retrieve results of join queries in O(n) relative to O(n2) in traditional relational database. Our model is extremely scalable and has been tested with tens of GigaBytes of data
