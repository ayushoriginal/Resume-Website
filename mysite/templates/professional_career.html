✔ Project : Design and Implementation of Cloud Data Synchronization on Android Platform

✔ Mentor : Rochan S


  • Project Details

    My team consisted of 2 people. Our task was to implement methods to synchronize data between an android client and central server. Data on Android clients was stored using SQLite and on the server both MongoDB and MySQL was used. Both kinds of data was considered - Master Data (once single copy on server for all clients) as well as Transaction Data (one corresponding copy on server for each client). The objective was achieved using various audit fields. My main contributions were :

    • Carried out a brief literature survey on best practices and available implementations
    • Designed a server-side conflict resolution mechanism based on audit fields
    • Optimization of synchronization mechanism for slow and erratic network by compressing data
    • Developed an efficient mechanism to schedule synchronization
    • Batching Network as well as I/O Operations
    • Measured performance under various simulated network condition (latency and rate of data transfer)
    • New mechanism was found to be 36% faster and consumed 23% less bandwidth

✔ Project : Recommendation System for Adaptive E-Learning

✔ Mentor : Dr. Deepak B Phatak, Nagesh Karmali


  • Research and Implementation Details

    My team consisted of 4 people - all 4 from different engineering institute in India. The main objective was to improve the way a student learns new concepts via an E-Learning platform (EdX and Moodle in particular). We realized than an effective E-Learning platform needs to adapt itself to the user's need. We explored the various ways of collecting, storing and finally utilizing useful information about the user, instructor and learning material to aid the process of personalization. My main responsibilities were :

    • Examined the Data available on Moodle and EdX
    • Designed and Implemented a new user model to model a user in terms of data
    • Developed a semi-automatic concept mapping mechanism using question-concept relationship
    • Implemented popular algorithms like Collaborative Filtering and Co-Sine Similarity to find similar concepts
    • Developed a Directed Graph based polynomial-time algorithm to recommend next best concept to user
    • Implemented a feedback mechanism and a Pershiva based System Evaluation with Recall and Precision parameters
    • The new recommendation feature was found useful by 78% user decreased average course completion time by 30%

    All implementation was done using a django-app so that it can be easily integrated with EdX which is also powered by Django. We used tools like Celery and RabbitMQ to queue computationally demanding items. Standard MySQL was used as the database. A nice GUI was developed using bootstrap and JQuery. More details about our implementation can be found in the project report.
